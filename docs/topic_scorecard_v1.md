# 选题评估打分卡 v1

> 基于 321 篇爆款（929~34.8 万赞）+ 32 篇不爆（4~248 赞）的对比分析。
> 满分 100 分，得分越高，万赞概率越大。

建议阈值：
- ≥50 分：立刻做
- 35-49 分：值得做，排进计划
- 20-34 分：看执行能力能否补分
- <20 分：谨慎或跳过

## A. 选题品类（0~30 分）

选题本身属于什么类型，决定天花板高低。

| 品类 | 基础分 | 判断依据 |
| --- | ---: | --- |
| 单产品/技术叙事 | 30 | 围绕一个具体新产品、新技术、新发明展开，有实物或演示画面 |
| 事件速报 | 28 | 某公司/团队刚刚发布/公布了什么，强时效性 |
| AI趋势/观点 | 25 | 围绕 AI 行业变化、趋势判断、观点洞察 |
| 多工具/资源合集 | 22 | 标题含 N 个工具/网站/神器，多产品并列 |
| 资源分享 | 20 | 推荐网站/平台/免费资源，不含操作教学 |
| AI/设计小技巧演示 | 18 | 小技巧、强演示、依赖画面呈现 |
| 效率工具推荐 | 17 | 推荐提升效率的工具或工作流 |
| AI出图/设计 | 16 | AI 图像、设计、视觉生成相关 |
| AI+行业应用 | 15 | AI 在具体行业里的应用，受众较窄 |
| 评测体验 | 15 | 体验、测评、上手反馈 |
| AI视频制作 | 12 | AI 视频工具/制作，竞争红海 |
| 教程/操作指南 | 10 | 标题含教你/怎么做/教程/手把手，重操作步骤 |
| 开发者工具 | 8 | 面向程序员/技术受众，受众窄 |

## B. 信息差程度（0~25 分）

对目标受众来说，这个信息有多新。

| 信息差等级 | 分数 | 判断标准 |
| --- | ---: | --- |
| 全球/全网首发 | 25 | 世界第一个、全球首款、从未有人做过 |
| 时效首发 | 20 | 刚刚发布、昨天/今天宣布、深夜突发 |
| 海外搬运 | 12 | 来源是海外、Reddit、GitHub、外网，国内少人知道 |
| 认知差 | 8 | 很多人不知道、隐藏功能、你可能没见过 |
| 已知信息 | 0 | 已被多人报道过，或公开已久 |

## C. 爆款信号加分（上限 30 分）

0 个信号万赞率约 30%，1 个约 54%，2 个约 88%，3 个约 100%。

| 信号 | 分数 | 判断标准 |
| --- | ---: | --- |
| 有人物/团队故事 | +10 | 提到创始人、研究团队、具体人物经历，有人的故事可讲 |
| “终于/第一”型选题 | +10 | 标题含终于、第一个、首次、首款，满足长期等待情绪 |
| 反常识/意外性 | +10 | 品牌或人物做了意想不到的事，颠覆认知 |
| 有明确价格 | +8 | 正文提到具体价格数字，给出可触达感 |

## D. 受众与情绪修正（-8 ~ +5 分）

| 维度 | 修正 |
| --- | ---: |
| 受众=垂直人群 | -8 |
| 情绪=震撼/惊叹 | +5 |
| 情绪=好奇/趣味 | +5 |
| 情绪=未来感/想象 | +5 |
| 情绪=焦虑/实用 | +3 |
| 情绪=白嫖/占便宜 | +3 |
| 无明确情绪钩子 | +0 |

## E. 可触达感修正（0 ~ +5 分）

- 多工具/资源合集 + 有可触达感：+5
- 资源分享 + 有可触达感：+5
- 其他品类：不加不减

注意：可触达感是合集/资源类的必要条件，不是充分条件。

## 人设匹配加成

- 科技叙事体：单产品/事件速报品类 +5
- 效率清单体：多工具合集/资源分享品类 +5

## 局限性

打分卡主要解释选题结构层面的潜力，不能完全解释：
- 账号粉丝基数和人设
- 执行质量（文案、画面、剪辑）
- 发布时机和算法推荐
- 内容格式（短视频演示、口播、图文）

因此它适合作为自动化监控系统的初筛和优先级排序，不应作为唯一判断。
